# Critic Output Format

## HUMAN_REVIEW_NEEDED.md — Survey Assessment Section

```markdown
## Survey Assessment

### Weak Arguments (need bolstering)
- [Section]: [claim] — evidence is [issue]. Suggestion: [fix]

### Contradictions (human should weigh in)
- [Paper A] vs [Paper B] on [topic]: [nature of contradiction]

### Structural Suggestions
- [Sections to merge/split/reorder with reasoning]

### Figure Proposals (approve/reject each)
- [ ] **Fig 1: [Title]** — [Type], data from [sources]. Purpose: [1 sentence]
- [ ] **Fig 2: [Title]** — [Type], data from [sources]. Purpose: [1 sentence]

### Ready for Writing?
[Assessment of whether the corpus + analysis is sufficient to begin writing]
```

## figure_proposals.md — Figure Proposal Mode

Output file: `AI-generated-outputs/<thread>/critic-review/figure_proposals.md`

Each proposal is a self-contained entry with enough detail for research-coder to implement without re-reading the deep-reader reports.

```markdown
# Figure Proposals

Generated by: critic (FIGURE-PROPOSAL mode)
Date: [YYYY-MM-DD]
Thread: [thread]

---

## Proposal 1: [Descriptive Title]

**Impact:** [HIGH / MEDIUM / LOW]
**Type:** [bar chart / line plot / scatter plot / heatmap / table-figure / schematic / comparison panel]
**Supports claim:** "[Exact claim text from manuscript or planned section]"
**Section:** [Section number or name where this figure would appear]

### Data Sources
- **Source 1:** [Author Year] — [what data to extract]. Location: `AI-generated-outputs/<thread>/deep-analysis/[paper]/notes.md`, section "[section name]"
- **Source 2:** [Author Year] — [what data to extract]. Location: [path]

### Data Points (if available)
| [Column 1] | [Column 2] | [Column 3] | Source |
|-------------|-------------|-------------|--------|
| [value]     | [value]     | [value]     | [key]  |

*If exact numbers are not yet extracted, state what needs to be extracted and from where.*

### Why This Figure
[1-2 sentences: why visual representation is more effective than text for this claim. What comparison or pattern becomes visible?]

### Design Notes
[Optional: suggested layout, axis labels, key comparisons to highlight. Keep brief — research-coder and figure-stylist handle details.]

---

## Proposal 2: [Descriptive Title]
...
```

### Impact Level Criteria

- **HIGH:** Directly supports the paper's central thesis or a novel contribution. Reader understanding significantly improves with visual representation. Quantitative comparison across 3+ sources.
- **MEDIUM:** Clarifies a secondary argument or comparison. Helpful but paper is comprehensible without it. Comparison across 2 sources.
- **LOW:** Illustrative or pedagogical. Improves reader experience but does not carry argumentative weight. Single-source data visualization.

## Commit Gates

All modes must pass before committing:

1. **Survey assessment:**
   - `figure_proposals.md` exists and contains at least one proposal (or explicit "No figures needed" statement)
   - `report.tex` exists and is non-empty
   - `HUMAN_REVIEW_NEEDED.md` contains `## Survey Assessment` header

2. **Style check:**
   - `HUMAN_REVIEW_NEEDED.md` contains `## Style Check —` header with section name
   - Verdict line present (PASS or REVISE)

3. **Journal compliance:**
   - `HUMAN_REVIEW_NEEDED.md` contains `## Journal Compliance Check` header
   - Word count and page limit lines present

4. **Figure compliance:**
   - `HUMAN_REVIEW_NEEDED.md` contains `## Figure Compliance Check` header
   - At least one figure entry present

5. **Figure proposal:**
   - `figure_proposals.md` exists in `AI-generated-outputs/<thread>/critic-review/`
   - Each proposal has: Impact level, Type, Data Sources section, "Why This Figure" section
   - `HUMAN_REVIEW_NEEDED.md` contains `## Figure Proposals` header
   - Each proposal listed with checkbox for human approval
